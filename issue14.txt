## Technical Notes

**next.config.ts — must have both:**
```typescript
const nextConfig: NextConfig = {
  output: 'standalone',
  serverExternalPackages: ['better-sqlite3'],
}
```

**Dockerfile:**
```dockerfile
FROM node:22-alpine AS deps
WORKDIR /app
COPY package.json package-lock.json* ./
RUN npm ci

FROM node:22-alpine AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN npm run build

FROM node:22-alpine AS runner
WORKDIR /app
ENV NODE_ENV=production
RUN addgroup --system --gid 1001 nodejs && adduser --system --uid 1001 nextjs
COPY --from=builder /app/public ./public
COPY --from=builder --chown=nextjs:nodejs /app/.next/standalone ./
COPY --from=builder --chown=nextjs:nodejs /app/.next/static ./.next/static
RUN mkdir -p /app/data && chown nextjs:nodejs /app/data
USER nextjs
EXPOSE 3000
ENV PORT=3000 HOSTNAME="0.0.0.0"
CMD ["node", "server.js"]
```

**Note on native module:** If `npm ci` fails in Alpine due to better-sqlite3, add `RUN apk add --no-cache python3 make g++` before `npm ci` in the deps stage. Note this in the PR description if you add it.

**docker-compose.yml:**
```yaml
services:
  news-hub:
    build: .
    restart: unless-stopped
    ports: ["3001:3000"]
    environment:
      - ADMIN_PASSWORD=${ADMIN_PASSWORD}
      - DATABASE_URL=/app/data/news.db
      - NEXT_PUBLIC_SITE_URL=${NEXT_PUBLIC_SITE_URL}
      - NEXT_PUBLIC_X_HANDLE=${NEXT_PUBLIC_X_HANDLE}
    volumes: ["./data:/app/data"]
```

**nginx.conf:** Proxy to `localhost:3001`. Headers: `Host $host`, `X-Real-IP $remote_addr`, `X-Forwarded-For $proxy_add_x_forwarded_for`, `X-Forwarded-Proto $scheme`. TLS: certs at `/etc/letsencrypt/live/news.ernestofgaia.xyz/`.
